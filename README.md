# 🛡️ VaaniRekha — Multilingual AI Voice Fraud Detection API

> **Built for the GUVI × HCL Hackathon** | Powered by [Sarvam AI](https://sarvam.ai) & Meta Wav2Vec2

---

## 📌 Overview

**VaaniRekha** is a production-ready REST API that detects AI-generated (deepfake) voices and identifies financial fraud in phone call audio — across **English, Hindi, Tamil, Telugu, Malayalam, Bengali, Kannada, Marathi, and Gujarati**.

It combines two detection strategies in a single, low-latency pipeline:

1. **AI Voice Detection** — Identifies whether speech was generated by a text-to-speech (TTS) engine using a fine-tuned deepfake audio classifier based on Meta's `wav2vec2-large-xlsr`.
2. **Fraud Keyword Detection** — Transcribes the audio using **Sarvam AI's `saaras:v3`** multilingual ASR model, translates the transcript to English using **Sarvam's `mayura:v1`** translation model, and scans for high-risk fraud keywords (OTP, Aadhar, PAN, lottery, CVV, etc.) in both the native language and the English translation.

The combined result produces a final `fraud` verdict with a risk tier (`HIGH`, `MEDIUM`, or `LOW`) and a human-readable `explanation`.

## 🚀 Key Features

| Feature | Detail |
|---|---|
| **AI Voice Detection** | wav2vec2-large-xlsr deepfake classifier — language-agnostic |
| **Multilingual STT** | Sarvam AI `saaras:v3` — auto-detects 9+ Indian languages |
| **Automatic Translation** | Sarvam AI `mayura:v1` — translates to English for keyword scanning |
| **Dual-Segment Transcription** | First 15s + Last 15s of a call processed in parallel to catch late-call fraud cues |
| **Metadata Short-Circuit** | Instantly flags audio encoded by FFmpeg/Lavf (used by TTS APIs) before model inference |
| **Tiered Risk Scoring** | `HIGH` / `MEDIUM` / `LOW` risk based on keyword weight and count |
| **Sub-6s Latency** | All operations — STT, translation, and model inference — designed to complete within 6 seconds |
| **Dockerized & Railway-Ready** | One-command deployment via Docker or Railway |

## 🏗️ System Architecture

```
Audio Input (Base64 / URL)
        │
        ▼
┌─────────────────────────────────────────────────────────────┐
│                     FastAPI Application                      │
│                                                             │
│  ┌─────────────────────┐    ┌──────────────────────────┐   │
│  │  Voice Detector     │    │      Sarvam AI Client    │   │
│  │  (CPU-bound)        │    │      (I/O-bound, async)  │   │
│  │                     │    │                          │   │
│  │  1. Load Audio      │    │  1. STT: first 15s       │   │
│  │  2. Preprocess      │◄──►│  2. STT: last 15s        │   │
│  │     (16kHz mono)    │    │     (parallel requests)  │   │
│  │  3. Metadata check  │    │  3. Translate → English  │   │
│  │  4. Wav2Vec2 model  │    │                          │   │
│  │  5. Pitch heuristic │    │  API: sarvam.ai/speech-  │   │
│  │     (if uncertain)  │    │       to-text            │   │
│  └──────────┬──────────┘    └──────────┬───────────────┘   │
│             │                          │                    │
│             ▼                          ▼                    │
│         AI Prob             Transcript + Language           │
│             │                          │                    │
│             └──────────┬───────────────┘                    │
│                        ▼                                    │
│              Keyword Analysis (native + English)            │
│                        │                                    │
│                        ▼                                    │
│              Risk Scoring: HIGH / MEDIUM / LOW              │
│                        │                                    │
│                        ▼                                    │
│                  Fraud Verdict + JSON Response              │
└─────────────────────────────────────────────────────────────┘
```

The `VoiceDetector` (CPU-bound) and all Sarvam STT requests (I/O-bound) run **concurrently** using `asyncio.create_task` + `loop.run_in_executor`, ensuring minimum end-to-end latency.

## 🤖 Sarvam AI Integration

This project uses **two Sarvam AI APIs**:

### 1. Speech-to-Text — `saaras:v3`
- **Endpoint:** `https://api.sarvam.ai/speech-to-text`
- **Purpose:** Transcribes phone call audio in any of 9+ Indian languages, auto-detecting the spoken language.
- **Strategy:** For audio longer than 30 seconds, the **first 15 seconds** and **last 15 seconds** are sent as parallel async requests. This ensures fraud keywords that appear late in a call (e.g., "OTP", "password") are always captured, without processing the full audio.

### 2. Text Translation — `mayura:v1`
- **Endpoint:** `https://api.sarvam.ai/translate`
- **Purpose:** Translates non-English transcripts to English (formal mode), enabling English-based keyword matching on any Indian language call.
- **Supported source languages:** `hi-IN`, `ta-IN`, `te-IN`, `ml-IN`, `bn-IN`, `kn-IN`, `mr-IN`, `gu-IN`

Both clients enforce strict timeouts (4.5s for STT, 2.0s for translation) to keep the total API response time within acceptable limits.

## 🔍 Detection Logic

### Stage 1 — Metadata Short-Circuit (Instant)
Audio files encoded by programmatic tools (FFmpeg, Libavformat) carry encoder metadata tags like `Lavf` or `lavc`. Since real human recordings from phones/microphones never carry these tags, a match triggers an immediate `+0.20` boost to the AI probability before model inference even runs.

### Stage 2 — AI vs. Human Classification (Wav2Vec2)
The model `Gustking/wav2vec2-large-xlsr-deepfake-audio-classification` classifies audio as `real` (label 0) or `fake` (label 1). It is evaluated on the first **2 seconds** of audio (post-preprocessing) for sub-200ms inference latency.

### Stage 3 — Pitch Heuristics (Conditional)
Only runs when the model is **uncertain** (predicted AI probability between 0.35–0.65). Analyzes:
- **Pitch Standard Deviation** — AI voices are monotone; real voices have rich intonation.
- **Pitch Jitter** — Human voices have natural micro-variations (2–8 Hz sweet spot); TTS voices are either too smooth (<1 Hz) or noisy (>12 Hz).

### Stage 4 — Fraud Risk Assessment
After keyword scanning, a tiered decision is made:

| Risk Level | Condition |
|---|---|
| `HIGH` | 2+ high-risk keywords, OR 1 high + 2 low |
| `MEDIUM` | 1 high-risk keyword, OR 4+ low-risk keywords |
| `LOW` | Any remaining keyword matches |

> **Note:** An AI-generated voice alone does **not** flag a call as fraud. It bumps the risk from `LOW → MEDIUM` as a warning signal. Fraud requires keyword evidence.

## 📡 API Reference

All endpoints require an `x-api-key` header.

### `POST /detect`
Full analysis endpoint. Accepts audio as Base64 or a public URL.

**Request Body:**
```json
{
  "audio_base64": "<base64-encoded audio string>",
  "audio_url": "https://example.com/call.mp3"
}
```
*(Provide either `audio_base64` or `audio_url` — not both.)*

**Response:**
```json
{
  "classification": "AI",
  "confidence_score": 0.92,
  "ai_probability": 0.92,
  "detected_language": "hi-IN",
  "transcription": "आपका बैंक खाता ब्लॉक हो जाएगा, अभी OTP शेयर करें",
  "english_translation": "Your bank account will be blocked, share OTP now",
  "fraud_keywords": ["otp (english) [HIGH]", "blocked (english) [HIGH]", "bank (english) [LOW]"],
  "overall_risk": "HIGH",
  "fraud": true,
  "confidence": 0.95,
  "explanation": "AI probability 0.92, Deepfake detector classified as AI, HIGH RISK — 2 critical + 1 context keywords: otp, blocked, bank",
  "audio_duration_seconds": 6.0,
  "num_chunks_processed": 1,
  "chunk_ai_probabilities": [0.92]
}
```

### `GET /detect?audio_url=<url>`
Convenience GET wrapper for the POST `/detect` endpoint. Same response format.

### `POST /api/voice-detection`
Strict hackathon-format endpoint.

**Request Body:**
```json
{
  "language": "hi-IN",
  "audioFormat": "mp3",
  "audioBase64": "<base64-encoded mp3>"
}
```

**Response:**
```json
{
  "status": "success",
  "language": "hi-IN",
  "classification": "AI_GENERATED",
  "confidenceScore": 0.92,
  "explanation": "AI probability 0.92, Deepfake detector classified as AI"
}
```

### `GET /health`
Returns `{"status": "ok"}` — use for uptime monitoring.

## ⚙️ Local Setup

### Prerequisites
- Python 3.10+
- `ffmpeg` installed (`brew install ffmpeg` on macOS)

### 1. Clone and install dependencies
```bash
git clone <your-repo-url>
cd Multilingual-guvi-hcl-main
pip install -r requirements.txt
```

### 2. Configure environment variables
Create a `.env` file in the project root:
```env
API_KEY=your_secret_api_key
SARVAM_API_KEY=your_sarvam_api_key
```

Get your Sarvam API key from [https://dashboard.sarvam.ai](https://dashboard.sarvam.ai).

### 3. Run the server
```bash
uvicorn app.main:app --host 0.0.0.0 --port 8000 --reload
```

The API will be available at `http://localhost:8000`. The Wav2Vec2 model (~1.2 GB) is downloaded from Hugging Face on first startup.

## 🐳 Docker Deployment

```bash
# Build the image
docker build -t vaanirekha .

# Run with environment variables
docker run -p 8000:8000 \
  -e API_KEY=your_secret_key \
  -e SARVAM_API_KEY=your_sarvam_key \
  vaanirekha
```

> The Hugging Face model is downloaded at **runtime** (not build time) because Railway restricts outbound internet access during Docker builds.

## 🚂 Railway Deployment

1. Push this repository to GitHub.
2. Create a new Railway project and connect your GitHub repo.
3. Add the following environment variables in the Railway dashboard:
   - `API_KEY` — your chosen secret API key
   - `SARVAM_API_KEY` — your Sarvam AI API subscription key
4. Railway will auto-detect the `Dockerfile` and deploy.

The `railway.json` file is pre-configured with health check settings.
